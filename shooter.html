Unity C# 個人專案

專案簡介

以 Unity Learn 的 Junior Programmer Path 課程作為起點， 從設計文件、幾何原型到最終調整，完整走過一次獨立開發流程。

遊戲概念參考經典街機射擊遊戲「Raiden」，玩家在場景中移動並射擊敵人， 透過拾取限時強化道具切換攻擊模式。 開發過程中發現原版垂直畫面不適合 PC， 因此自行調整為橫向視角並重新設計操作方式。

核心實作

  • 玩家控制： WASD 全向移動搭配滑鼠瞄準，射擊時機身自動朝向游標方向旋轉
  • Spawn Manager： 統一管理敵人與道具的生成時機與位置，遊戲節奏可集中調整
  • Object Pooling： 子彈與敵人改為預先建立並重複使用，避免頻繁 Instantiate / Destroy 造成效能波動
  • 強化系統： 拾取道具後切換攻擊模式，效果持續 10 秒；效果期間再次拾取會重置倒數

設計與技術挑戰

操作方式的設計取捨

原本照著 Raiden 的垂直捲軸設計，但實際在 PC 上看 9:16 的畫面比例就覺得不對， 於是改成橫向視角並讓玩家可以全方向自由移動。

操作方式隨之要重新設計。試過幾個方案：維持單方向敵人進攻、 用鍵盤做左右旋轉加前進、改用滑鼠控制射擊方向。 實際玩下來，WASD 移動搭配滑鼠瞄準的手感最直覺，反應也最快， 最終採用這個方案。

強化剩餘時間的顯示設計

強化道具有 10 秒的持續時間，需要讓玩家在戰鬥中隨時感知還剩多久。 一開始做成血量旁的 icon，以類似線上遊戲冷卻的方式顯示倒數。

但實際遊玩時發現，戰鬥中視線幾乎都鎖在機身附近， 旁邊的 icon 根本不會去看。

最後改成在機身上疊一層半透明的藍色光暈， 隨時間逐漸黯淡消散。玩家不需要移開視線， 就能透過光暈的強弱感受到剩餘時間。

頻繁生成物件帶來的效能問題

子彈與敵人在遊戲中會持續生成與銷毀， 初期使用 Instantiate / Destroy 實作， 物件數量增加後出現明顯的效能波動。

改為 Object Pooling 後，預先建立一批物件並重複使用， 避免執行期間頻繁分配與釋放記憶體， 在高頻率射擊與敵人生成下仍能維持穩定的執行表現。

開發收穫

這是第一個從頭到尾自己規劃的專案。 最大的收穫是理解到設計不是一次到位的——很多決定要實際玩過才知道哪裡不對， 像是畫面比例、操作方式、視覺回饋的位置，都是玩了之後才發現問題、再去調整。 比起課程本身教的內容，這個「發現問題再修正」的過程讓我學到更多。
