Unity C# 個人專案

專案簡介

以 Unity Learn 的 Junior Programmer Path 課程作為起點， 從設計文件、原型建立到最終調整皆由個人完整實作。

遊戲概念參考經典街機射擊遊戲「Raiden」，玩家在場景中移動並射擊敵人， 並透過拾取強化道具提升攻擊能力。

負責內容

  • 遊戲系統設計與實作
  • 玩家操作與射擊機制設計
  • 敵人與道具生成邏輯
  • 遊戲節奏與體驗調整

核心實作

  • 玩家控制與射擊系統： 採用 WASD 移動搭配滑鼠射擊，並實作角色朝向滑鼠方向轉動
  • 敵人與生成機制： 透過 Spawn Manager 統一管理敵人與道具的生成時機與位置， 使遊戲節奏能夠集中調整與控制
  • 物件管理（Object Pooling）： 將子彈與敵人物件改為重複使用，避免頻繁 Instantiate / Destroy， 提升執行效率並降低效能波動
  • 強化系統： 玩家可透過拾取道具改變攻擊模式，提升火力與遊戲變化性
  • 畫面與遊玩模式調整： 將原本垂直卷軸改為符合 PC 的橫向視角，並調整整體操作方式

設計與技術挑戰

操作方式的設計取捨

在將遊戲從傳統垂直卷軸改為自由移動後，原本的操作方式不再適用， 需要重新設計玩家的移動與射擊邏輯，以確保操作流暢。

過程中嘗試了多種方案，包括維持單方向敵人進攻、使用鍵盤旋轉控制， 以及滑鼠射擊等方式，並透過實際遊玩體驗比較其操作手感。

最終選擇以滑鼠作為射擊方向，搭配 WASD 移動， 在操作直覺性與反應速度之間取得較佳平衡。

頻繁生成物件帶來的效能問題

在遊戲中，子彈與敵人會持續生成與銷毀， 初期使用 Instantiate / Destroy 實作時， 在物件數量增加後出現效能波動的情況。

為了改善此問題，將相關物件改為使用 Object Pooling， 預先建立並重複使用物件， 避免執行期間頻繁分配與釋放記憶體。

透過此方式，使遊戲在高頻率射擊與敵人生成的情況下， 仍能維持穩定的執行表現。
