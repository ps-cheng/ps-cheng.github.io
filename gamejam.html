Hello, Godot! Game Jam Experiment 2026 專案

專案簡介

於 Game Jam 中 48 小時內完成的 2D 平台遊戲，大會主題為「Break the Loop」。
玩家需在重複的工作日常中探索這間黑心公司並逃離循環。

負責內容

  • 關卡設計與遊戲概念發想
  • 敵人行為實作（兩種敵人）
  • 互動物件（傳送門）
  • 場景圖塊（Tile）配置與關卡建置

核心實作

  • 關卡與場景系統： 使用 Tile-based 方式建構關卡，並依據遊戲節奏安排敵人與互動物件配置。
  • 敵人行為： 實作基本行為邏輯，依據玩家位置觸發互動。
  • 互動物件： 以 trigger 為基礎設計互動機制，包含傳送門等功能物件。

技術挑戰

平行開發下的測試問題

在開發過程中，關卡與敵人系統需要與玩家互動，但玩家控制由其他組員負責， 無法等待完整實作完成後再進行測試。

為了維持開發進度，我建立了一個簡化的玩家測試物件， 僅保留基本移動功能，並先以碰撞方式觸發互動。 在最終整合時，再將互動邏輯改為正式的操作觸發（按鍵互動）。

這樣的做法讓關卡與物件系統可以獨立開發與驗證， 降低對其他模組完成度的依賴。

多人協作的程式整合

團隊由多名程式組成，且在初期未建立統一的系統架構， 導致各模組之間的整合成本較高。

在時間有限的情況下，我們選擇先以模組為單位各自開發， 並透過 GitHub 進行版本控管，將不同功能拆分在各自資料夾中， 再由一名組員負責最終整合。

雖然這種方式在架構上較為鬆散， 但能有效降低開發期間的衝突， 並在 Game Jam 的時間壓力下維持開發進度。

開發收穫

  • 理解多人協作下模組拆分與整合的重要性
  • 學習在時間壓力下優先確保可遊玩性
  • 累積與美術、音效協作的經驗
