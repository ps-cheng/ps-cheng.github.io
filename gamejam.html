Hello, Godot! Game Jam Experiment 2026

Godot GDScript 48-Hour Jam 合作專案

專案簡介

第一次參加 Game Jam，也是第一次使用 Godot 開發。參加前透過 YouTube 教學自學引擎基礎， 與三名程式和一名 UI/UX 組成五人團隊，在 48 小時內完成。

遊戲概念由我提出——以大會主題「Break the Loop」為出發， 設計成一名被困在辦公室日常迴圈中的上班族試圖逃脫的故事。

負責內容

  • 遊戲概念與故事設定發想
  • 關卡設計與 Tile 地圖建置
  • 巡邏型敵人行為實作
  • 傳送門系統（與組員模組對接）

核心實作

  • 巡邏敵人： 實作左右徘徊邏輯，玩家碰撞時觸發 Game Over。
  • 傳送門： 負責接收互動事件並將玩家位置切換至對應出口，與負責互動提示的組員分工對接。
  • 關卡設計： 以 Tile 建構場景，依遊戲節奏安排敵人分佈與互動物件配置。

技術挑戰

平行開發下的測試問題

關卡與敵人系統需要與玩家物件互動，但玩家控制由其他組員負責， 無法等待對方完成再測試。

為了維持進度，我建立了一個簡化的測試用玩家物件， 只保留基本移動功能，互動也先以碰撞觸發代替按鍵。 整合時再由對應組員將觸發邏輯改回正式版本。

這樣的做法讓我的模組可以獨立驗證， 不需要依賴其他部分的完成度。

多人協作的程式整合

團隊在初期未建立統一架構，五個人各自開發不同模組， 整合時的對接成本比預期高。

在時間壓力下，我們選擇先以模組為單位各自推進， 透過 GitHub 分資料夾進行版本控管， 最後由一名組員統一將各模組整合進主場景。

開發收穫

第一次在時間壓力下多人協作，實際體會到「先讓自己的模組可測試、再求整合」的開發節奏。 也讓我理解事先對齊介面與資料格式的重要性——等到要整合才發現沒對齊，代價會很高。
